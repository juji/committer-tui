# v1.0 Readiness

## Core flows (all must work without crashes)

- [ ] **Commit flow**: select files → generate message → confirm → commit success
- [ ] **Commit with no files**: shows error, "Redo" button focused
- [ ] **All models fail**: shows error message properly, "Redo" available
- [ ] **Config**: add/edit/remove/reorder provider, change instructions, save, reopen — values persist
- [ ] **Config with no models**: forces config popup, can't proceed without at least one
- [ ] **History**: browse entries, view diffs, ESC back, keyboard navigation
- [ ] **Theme**: change theme, close config, reopen — selection persists
- [ ] **Splash → app**: valid keys auto-proceed, partial shows buttons, all-failed shows buttons
- [ ] **Edit message**: modify generated message before committing, ESC to go back

## Error handling (no silent failures)

- [ ] Network failure during API key validation — graceful message, not a crash
- [ ] Network failure during message generation — fallback to next model, final error shown
- [ ] Malformed config file — detected, user sent to config screen
- [ ] Missing config file — created on first save
- [ ] Git not in PATH — meaningful error, not a cryptic thrown exception
- [ ] Git repo not found — meaningful error
- [ ] Terminal too small — graceful handling, no layout breakage or crash
- [ ] Double-enter on Generate/Confirm/Commit — no duplicate operations

## State machine integrity

- [ ] Commit store resets fully between flows (no stale message/error/files)
- [ ] `generating` and `committing` guards prevent parallel operations
- [ ] All keyboard scopes properly popped on unmount (no ghost listeners)
- [ ] Popup open/close idempotent (closing when already closed is safe)
- [ ] Focus never gets "lost" (no state where no element is focused)

## Polish

- [ ] All async operations show some loading indicator
- [ ] Error messages are human-readable (no stack traces in UI)
- [ ] Keyboard shortcuts documented somewhere (`ctrl+g` config, `ctrl+y` history, etc.)
- [ ] Long file paths / commit messages don't break layout
- [ ] Color scheme works in light terminals (or at least doesn't crash)
- [ ] `--help` flag or usage text

## When to call it v1.0

When the above checklist is fully green and you've dogfooded it for your own commits
for at least a week without hitting an unhandled error or getting confused about what
to do next. v1.0 means "I'd tell a colleague to use this."
