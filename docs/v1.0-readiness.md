# v1.0 Readiness

## Core flows (all must work without crashes)

- [x] **Commit flow**: select files → generate message → confirm → commit success
- [x] **Commit with no files**: shows error, "Redo" button focused
- [x] **All models fail**: shows error message properly, "Redo" available
- [x] **Config**: add/edit/remove/reorder provider, change instructions, save, reopen — values persist
- [x] **Config with no models**: forces config popup, can't proceed without at least one
- [x] **History**: browse entries, view diffs, ESC back, keyboard navigation
- [x] **Theme**: change theme, close config, reopen — selection persists
- [x] **Splash → app**: valid keys auto-proceed, partial shows buttons, all-failed shows buttons
- [x] **Edit message**: modify generated message before committing, ESC to go back

## Error handling (no silent failures)

- [x] Network failure during API key validation — graceful message, not a crash
- [x] Network failure during message generation — fallback to next model, final error shown
- [x] Malformed config file — detected, user sent to config screen
- [x] Missing config file — created on first save
- [x] Git not in PATH — meaningful error, not a cryptic thrown exception
- [x] Git repo not found — meaningful error
- [x] Terminal too small — graceful handling, no layout breakage or crash
- [x] Double-enter on Generate/Confirm/Commit — no duplicate operations

## State machine integrity

- [x] Commit store resets fully between flows (no stale message/error/files)
- [x] `generating` and `committing` guards prevent parallel operations
- [x] All keyboard scopes properly popped on unmount (no ghost listeners)
- [x] Popup open/close idempotent (closing when already closed is safe)
- [x] Focus never gets "lost" (no state where no element is focused)

## Polish

- [x] All async operations show some loading indicator
- [x] Error messages are human-readable (no stack traces in UI)
- [x] Keyboard shortcuts documented somewhere (`ctrl+g` config, `ctrl+y` history, etc.)
- [x] Long file paths / commit messages don't break layout
- [x] Color scheme works in light terminals (or at least doesn't crash)
- [x] `--help` flag or usage text

## When to call it v1.0

When the above checklist is fully green and you've dogfooded it for your own commits
for at least a week without hitting an unhandled error or getting confused about what
to do next. v1.0 means "I'd tell a colleague to use this."
